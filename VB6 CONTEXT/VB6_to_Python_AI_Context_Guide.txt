AI MIGRATION CONTEXT GUIDE VB6 to Python (SQLite) Conversion Project

Project Background: This project involves migrating a legacy VB6
codebase into modern Python. The system is old and rarely modified. The
output files are sent to external partners. The goal is to preserve
functionality while improving readability and maintainability.

Supervisor Requirements: 1. Study and understand VB6 modules before
translating. 2. Perform incremental (part-by-part) migration. 3. Focus
on: - Database utilization (ADODB -> sqlite3) - File reading and
writing - Email sending functionality - Data conversion logic 4.
Optimize logic where appropriate using dictionaries. 5. Ensure the
Python code: - Is readable and maintainable - Uses parameterized SQL
queries - Closes database connections properly - Uses try/except for
error handling - Avoids unnecessary overengineering 6. Preserve original
behavior exactly.

Migration Rules: - Do NOT translate line-by-line blindly. - Translate
logic and intent. - Keep behavior identical unless explicitly improving
internal structure. - Separate responsibilities into logical Python
modules: main.py database.py models.py utils.py email_sender.py

For Each VB6 File: PHASE 1 – Analysis - Explain what the file does
section-by-section. - Identify database usage, file handling, email
logic, and data conversion. - Summarize execution flow.

PHASE 2 – Python Translation - Convert using sqlite3 for database. - Use
“with open” for file handling. - Use smtplib/email.message if email
logic exists. - Use dictionaries instead of long Select Case blocks. -
Add clear comments. - Ensure code is runnable.

PHASE 3 – Documentation Create separate documentation for each
translated module including: - Original VB6 file purpose - Python
equivalent file - Key functions - Database tables used - Inputs and
outputs - Improvements made - Notes for future maintainers

Coding Standards: - Clear function names - No duplicated logic - Proper
separation of concerns - Minimal global variables - Clean, structured
layout

Final Objective: Produce a clean, maintainable Python version of the
legacy VB6 system while preserving exact business functionality.
